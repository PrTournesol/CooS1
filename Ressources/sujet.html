Ce projet sera évalué et fera l’objet d’une note de contrôle continu. De plus, n’oubliez pas qu’il s’agit d’un module de conception et non de programmation.

1. Contexte

Le contexte de ce projet est celui de l’organisation des secours en cas d’incendie. On vous demande de réaliser un logiciel de simulation de l’utilisation de robots-pompiers OLE (cf. OLE).

Dans le cadre de ce projet, on imagine un système logiciel complexe permettant d’aider ls secours à combattre un ou plusieurs départs de feu sur une zone à risque délimitée et comportant :

  • un certains nombre d’OLE

  • un drone qui fournit une carte de la zone de risque

  • un manager (logiciel) qui assigne un objectif (feu) à un OLE

  • un pompier qui paramètre la simulation (carte, nombre et position des feux et des robots, vitesse de la simulation, etc.)

  • un ou plusieurs algorithmes (interchangeables) de calcul du plus court chemin

  • un simulateur permettant de tester le fonctionnement du manager

  • des interfaces de configuration des robots, des cartes, du manager, …​

2. Environnement de travail

Voici les conditions et contraintes du projet :

  • groupe de 3 ou 4 étudiants (6 projets par groupe de TD)

  • utilisation de redmine pour gérer les livraisons

    • Adresse des projets : http://algec.iut-blagnac.fr/redmine/projects/coo-projet-2016

  • utilisation de Subversion pour le versionnement

    • Un dépôt svn disponible pour chaque groupe : svn://algec.iut-blagnac.fr/data/COOS2/groupe[1-4]-[1-6]

  • utilisation de plantUML pour les modèles

  • utilisation de AsciiDoc pour la documentation

3. Attendus du projet

Plusieurs itérations (souvent appelés Jalons dans l’industrie) seront demandées.

3.1. Itérations

Chaque semaine correspondra à une itération, qui donnera lieu à une présentation de 5' par l’un des membres du groupe (dans la dernière 1/2 heure du dernier créneau de la semaine).

Chaque itération correspondra à une version de votre code et de sa documentation associée.

Grosso modo, les itérations suivent un "cycle en V" classique :

  1. Exigences et Analyse initiale

  2. Analyse et conception

  3. Developpement

5. Mise en place

5.1. Inscription des groupes projet sur Redmine

  1. Au sein de chaque groupe TD, constituer des groupes projets (4 étudiants max)

  2. Compléter la page Groupes Projet COO S2 2016

  3. Désigner un premier manager et indiquer son nom à votre enseignant pour qu'il le définisse dans le projet Redmine du groupe

  4. Le manager assigne le rôle de 'Développeur' à chaque membre de son équipe :

    1. 'Configuration' → 'Membres'

    2. Rechercher le membre à droite

    3. Cocher le rôle Développeur

    4. 'Ajouter'

Il est impératif que TOUS LES MEMBRES DE L’EQUIPE aient compris et soient capable de réaliser toutes les actions qui suivent à la fin de la 1ère semaine.

5.2. Accès aux dépôts svn

  1. Obtenez auprès de votre enseignant le login/password inital d'accès au dépôt svn de votre groupe

  2. Récupérer une copie de ce dépôt et modifiez le fichier groupeX-Y.pass en rajoutant une ligne pour chaque membre de votre groupe de la forme login = passwdsans oublier les espaces autour du signe =

  3. Commitez le fichier groupeX-Y.pass

5.3. Organisation du Code de l’application

  1. Créer les répertoires src et bin dans le dossier copie de votre dépôt svn

  2. Ajouter un fichier HelloMonde.java au répertoire src

    public class HelloMonde {

	/**
	 * @param args
	 */
	public static void main(String[] args) {
	     System.out.println("[DUT/INFO/S3]");
	     System.out.println("Hello Monde !");
	     System.out.println("depuis Blagnac.");
	}

}

  3. Compiler le fichier HelloMonde.java et placer HelloMonde.class dans le répertoire bin

    javac -cp . -d bin src\HelloMonde.java

5.4. Documentation

  1. Créer les répertoires tools , srcdoc et doc dans le dossier copie de votre dépôt svn

  2. Décompresser l’archive asciidoc-8.6.9.zip dans le répertoire tools\asciidoc869\

  3. Ajouter le fichier plantuml.jar dans le répertoire tools

  4. Ajouter le fichier diag0.puml au répertoire srcdoc

    '-----------------------------------
' UML concepts illustrated
' JMB 2016
'-----------------------------------
@startuml
'-----------------------------------

class CLIENTS {
	NOM
	PRENOM
	SOLDE
}

'-----------------------------------
@enduml
'-----------------------------------

  5. Décompresser l’archive images.zip dans le répertoire srcdoc\images\

  6. Produire le diagramme correspondant depuis le répertoire srcdoc

    cd srcdoc
java -jar ..\tools\plantuml.jar -Tpng -o images diag0.puml

  7. Ajouter le fichier doc.adoc au répertoire srcdoc

5.5. Makefile

  1. Ajouter le fichier make.bat à coté des répertoires src, srcdoc, tools, bin et doc

    rem echo off
set ASCIIDOCDIR=.\tools\asciidoc869\
set PLANTUMLDIR=.\tools\
set SPRINTDIR=%~dp0
set SRCDIR=.\src\
set BINDIR=.\bin\
set SRCDOCDIR=.\srcdoc\
set DOCDIR=.\doc\

set MAKETEST=1

@echo ///////////////////////////////////////////////////////
@echo // COMPILATION des executables
@echo ///////////////////////////////////////////////////////
javac -d %BINDIR% %SRCDIR%HelloMonde.java

@echo ///////////////////////////////////////////////////////
@echo // COMPILATION des documentations
@echo ///////////////////////////////////////////////////////
java -jar %PLANTUMLDIR%plantuml.jar -Tpng -o %SRCDOCDIR%images %SRCDOCDIR%diag0.puml
python %ASCIIDOCDIR%asciidoc.py -a data-uri -o %DOCDIR%doc.html %SRCDOCDIR%doc.adoc

  2. Exécuter le fichier : tout doit parfaitement fonctionner

Appendices

Frequently Asked Questions

Je mettrai dans cette section les réponses aux questions récurrentes qui seront posées aux clients.

Existe-t’il un prototype déjà réalisé?

Non, mais vous pourrez vous inspirez d’une réalisation récenterd’étudiants du Master ICE : http://kazhord.fr/gotorup.php.

Asciidoc et PlantUML sont-ils disponibles en salles TPs

Oui.

  • Vous pouvez ouvrir une console avec des raccourcis pour l'utilisation AsciiDoc et PlantUML depuis le menu "Démarrer / AsciiDoc / Console AsciiDoc"
  • Vous pouvez aussi utiliser l'éditeur AsciidocFX qui facilite l'édition des documents
  • Vous pouvez utilisez l'EDI Eclipse pour éditer des schémas PlantUML et SNI
  • Vous pouvez utilisez l'EDI IntelliJ IDEA pour éditer des schémas PlantUML et des documents AsciiDoc

La remises des versions (v0→v5) doit-elle se faire sous redmine ?

Oui sous deux formes : - un fichier (zip) dont le lien est clairement mentionné dans le wiki de votre projet - un répertoire vx à la racine de votre dépôt svn accessible depuis la rubrique dépôt de votre projet Redmine.

Les étudiants doivent-ils réellement développer une appli Java avec IHM ou seulement en faire la conception ?

L’IHM n’est pas obligatoire mais oui une application, même minimaliste, est attendu. Par contre il faut bien voir que ce programme compte assez peu dans la note finale.
